awesome_japanese_study_toolsの歩み

发布于 2022-09-08 | 分类于 日语 、 碎碎念 | 10分钟 | 2733字数 | 浏览量：

摘要：我希望只要是前人走过的弯路，后人就不必再走。这是我的信念，也是我建立这个项目的原因。也希望後輩们不会像我一样：用大把的时间来迷茫，在几个瞬间成长:)

2023-04-09

由于是不想学数学才选了日语，自己大学四年浪费时间走了不少弯路。要是大一时能有像 CS自学指南、上海交通大学生存手册 这样系统的指导手册，自己的大学四年也会更有意义吧。但遗憾的是，直到我毕业，也未能发现这样的项目。

好在早就预料到这个结果，所以自己一直都在计划搞一个这样的项目。不过，由于拖延症，直到大四才正式将 awesome_japanese_study_tools 这个项目正式提上了日程。本来只是打算模仿《上海交通大学生存手册》做一个日语系生存手册，但由于已决定不在国内读研究生，而且也不打算读纯日语相关方向的修士，所以今后将以「面向日语自学者」为主要目标群体进行维护和更新。

临近毕业，回头来看当初提的一大堆所谓建议，只有这句个人认为这个项目要想长久、高效地维护下去，应该鼓励更多的同学自发参与，而不是寄希望于几个人或是变成某个部门的任务算是是一矢中的：相比于受开源运动影响巨大的计算机行业，其他行业仍然处于「只想闭门造车」的阶段，但「你不奉献，我不奉献，谁来奉献?你也索取，我也索取，向谁索取?」

希望这个项目能帮到更多的人，也希望这个项目能鼓励更多人参与到「主动分享」的行动中来:)

2022-09-08

以下是系部建一个系部大群时，自己向相关负责人提出的一些建议，可以看做是awesome_japanese_study_tools这个项目开始的地方。

嗨！XX，很抱歉因为一些个人的想法来叨扰你哈，主要想就昨天下午新建的本科生交流群提几点不成熟建议：

  1. 我十分赞同建一个系级的大群，这样大家可以接触到更多日语系同窗；
  2. 但我个人认为 QQ 群不适合承担所有需求，比如：不适合沉淀那些有价值的讨论（手滑删了就没啦）、分享一些资料（奇奇怪怪的审核限制）、不方便分类整理、不方便同学参与（个人认为这个项目要想长久、高效地维护下去，应该鼓励更多的同学自发参与，而不是寄希望于几个人或是变成某个部门的任务）
  3. 所以我建议将 QQ 群讨论过有价值的内容和一些资料整理在其他平台。

Github

首推 GitHub，理由：（好吧，我就是在夹带私货 233）

  1. 审核宽松，虽然处于半墙不墙的状态，但大多数时候访问还是没问题的，实在不行，文字可以直接同步到 gitee（这个平台限制和 GitHub 唯一的明显区别就是，项目总大小不超过 1G）
  2. 没有上传文件的总大小限制，单个文件超过 100M 会有警告，但有其他办法可以解决
  3. 其他大学有类似的项目：比如 浙江大学课程攻略共享计划 ，上海交通大学生存手册，可以参考这些项目的架构和设计
  4. 参与项目十分“简单”，可以实现较为规范的多人协作——参与的同学只需要 3 步：Fork 一个分支-修改/上传文件-提交 PR；而管理的同学需要在提交 PR 之后，检查修改的部分（github 会显示修改了哪些地方），确认无误之后合并一下，其他同学就可以看到了（被驳回的 PR 会保存在专门的地方，并不会直接消失，也就是说不论是否审核通过，其他同学都可以看到，这样可以最大程度保证项目的透明度）
  5. issue 区可当做讨论区来讨论问题——与 QQ 消息不同，issue 的讨论会被记录归档，这样有些事情就不用反复回答

缺点

  1. 下载文件要 FQ，不然速度有时候会非常慢：建议收录的教材 PDF 在百度网盘和蓝奏云保存一份（后者在浏览器下载文件免登录不限速，但只能上传单个不超过 100M 的文件，不能嵌套分享文件夹）
  2. 大多数同学不熟悉这个工具，实际参与的欲望可能会非常低：这个……其实也没有那么难啦，如果是真的想参与这个项目的，应该可以学会吧（应该吧 233
  3. 需要下载到本地才能搜索：可以用 gitbook 提供在线搜索服务，免费，但架设和维护成本比较高，需要一定的技术手段（并不复杂）
  4. 管理项目的同学可能会稍微累一点，因为 GitHub 采用的技术是 git，可能需要熟练掌握 git 命令才能保证参与的同学提交的内容不会丢失和冲突，不过掌握 pull、push、merge、diff 这四个命令应该就可以了：这个工作可以交给技术部的同学（不好意思，未加入过技术部，不清楚技术部的同志是否愿意承担这样的重担），浙大的那个项目就是由浙大技术社团求实潮负责维护求是潮技术研发中心 (github.com)

语雀

其次语雀，理由：

  1. 多人协作机制较为完善（语雀本身就是阿里内部的知识管理工具），启用空间功能后，可以给予不同的人不同的权限——分工明确、流程清晰的话，项目的质量会有一定保障；
  2. 免费版可以上传 10G 的文件，下载不限速。考虑到项目的性质，应该可以申请到白嫖的公益计划 (yuque.com)
  3. 上手简单清晰，语雀的「团队」、「知识库」功能非常适合这个项目——会有很多文档和文件，单靠文件夹分类会比较辣眼睛吧。具体介绍可以参考这篇文章笔记软件哪家强？我来，飞书，还是语雀？ -空谷的回答 - 知乎
  4. 自带搜索、评论区功能，网页加载速度也很快

缺点

  1. 审核机制比较诡异；考虑到日语专业资料的特殊性，也许会被封号 233，不过我的一篇文章链接了很多敏感的国外网站，过了一年多屁事没有
  2. 即使申请到公益计划，可能也会有参与人数限制：不过最低的上限是 50 人，应该够用了 233

息流

然后推荐息流，理由：
1. 对于多人协作支持较为完善（反复强调这一点是为了防止有人恶意删除文件和保证项目的透明度），加载速度也很快
2. 下载不限速，只要有一个付费账号就可以没有限制上传文件（一年 60，教育版认证学生/教师个人专业版优惠折扣 (flowus.cn)）

缺点

  1. 白嫖是不可能白嫖了，免费版只能邀请 5 个人协作，最多上传 2G 的文件
  2. 只能按照文件夹划分文件，可能会比较混乱（语雀的「团队」、「知识库」可以很清楚的划分、整理文件），不过有搜索功能应该没有太大影响

其他不推荐的平台

然后就是一些个人不太推荐的平台：

  1. 腾讯文档：

    • 优点：可以和 QQ 群紧密连接，上手非常简单
      缺点：
    • 不能上传文件（虽然可以用百度网盘和蓝奏云链接）
    • 分类整理大量文档会比较麻烦
    • 跨文档搜索会比较麻烦
    • 多人协作没有审核和校对的环节（很容易引起纠纷）
    • 加载速度会被文章长度影响

  2. 印象笔记：（注：由于我很久没用印象笔记了，这个评价可能不是很准确哈）

    • 优点：很多同学和老师都比较熟悉，上手难度应该不大
    • 缺点：考虑到项目性质，可能要用印象 TEAMS | 印象笔记 (yinxiang.com)，而不是单纯的印象笔记，也就是说大家都不能白嫖

  3. 自建论坛：

    • 倒是不用从零开始写代码，选择Discuz或者Discourse套个壳就行了（才怪，多半还有一系列的问题还要解决 233）
    • 审核会宽松一些（但不可能是法外之地）
    • 架设和维护难度非常大，服务器成本一年至少得 500+
    • 主要还是受了这篇北语老师的文章我们坚持做教育是为了什么的启发
    • 你就当我浮夸吧.jpg233

我希望能够将这些隐晦的、不确定的、口口相传的资料和经验，变为公开的、易于获取的和大家能够共同完善、积累的共享资料。
